# 検討項目メモ（特に読み込み速度）

## 1. 読み込みの遅さについて（結論）
- 現状のボトルネックは **初回に全記事を取得してから描画している点** と、検索時に重い比較処理が走りやすい点。
- 今回、以下を実装して体感速度を改善。
  1. **ローカルキャッシュ優先表示**（最新確認はバックグラウンド）
  2. **検索入力のデバウンス**（220ms）
  3. **あいまい検索アルゴリズムを軽量化**（部分一致 + 順序一致）
  4. **検索用前処理テキストを事前計算**

## 2. スプレッドシート保存は継続してよいか
- 結論: **当面は可、ただし上限を決めるべき**。
- おすすめ運用:
  - 記事件数が少〜中規模（〜数万）: スプレッドシート + APIでも運用可能
  - 件数増加 / 読み込み遅延が続く: Cloudflare D1 + R2へ段階移行
- 最低でもAPI側で以下が必要:
  - ページング（`limit`, `cursor`）
  - 検索条件のサーバーサイド処理

## 3. 無限スクロールは継続してよいか
- 結論: **ニュース一覧では有効**。ただし併設推奨。
- 併設すべきもの:
  - 「ページトップへ戻る」
  - URLクエリで検索条件保持
  - 長文閲覧向けに「ページネーション表示」モード（将来）

## 4. デザインが古くないか
- 現在は紙面風に寄せたため、意図に合っている。
- 次段階の改善:
  - 見出し階層の統一（h1/h2/h3）
  - カード間余白とフォントサイズの微調整
  - ダークモード（任意）

## 5. 追加機能の実装優先度（提案）
1. **性能基盤**（最優先）
   - APIページング
   - CDNキャッシュ
   - 画像最適化（サイズ/フォーマット）
2. **ログイン本実装**
   - OAuth/Passkey
   - コメント/ポイントのサーバー保存
3. **お気に入り・既読の同期**
   - ローカル → サーバー同期
4. **あいまい検索強化**
   - サーバー側インデックス導入

- [ ] APIレスポンスに `ETag` または `Last-Modified` を追加

- [ ] 画像をCloudflare ImagesまたはR2+Transformで配信
